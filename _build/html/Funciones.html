Funciones¬∂

Las funciones son porciones de c√≥digo que nos sirven para modularizar nuestros programas y evitar en muchos casos la repetitividad de c√≥digo. De manera general una funci√≥n recibe algunos valores de entrada, los procesa y devuelve algunos valores de salida (o bien modifica algunas variables).

Funciones built-in¬∂

Python dispone de algunas funciones nativas que se cargan autom√°ticamente cuando se inicia el int√©rprete. Por ejemplo la funci√≥n max devuelve el mayor valor num√©rico de una lista de n√∫meros:

Tambi√©n existe una funci√≥n min, an√°loga a max:

Otro ejemplo de funci√≥n nativa es bin, la cual dado un n√∫mero en base 10 devuelve una cadena con la representaci√≥n en base 2.

Naturalmente, el valor devuelto por una funci√≥n se puede asignar a una variable y posteriormente ser utilizado:

Hay funciones que no devuelven como tal un valor, si no que pueden modificar directamente alguna variable global o simplemente mostrar algo en la salida est√°ndar como el caso de print.

Tendremos tambi√©n funciones que aceptan m√°s de un argumento, por ejemplo a la funci√≥n round podemos pasarle dos argumentos: un n√∫mero real y la cantidad de lugares decimales a considerar para el redondeo.

La funci√≥n input¬∂

En Python, la funci√≥n input permite obtener entrada por teclado, con la finalidad de tomar informaci√≥n proporcionada por el usuario. La sintaxis de input es:

Veamos un ejemplo:

Como puedes observar, el programa anterior solicita al usuario un valor de entrada utilizando el prompt Ingrese su nombre: ; en este caso el usuario ingres√≥ la cadena Jorge. Con la segunda l√≠nea √∫nicamente se toma el valor guardado en la variable nombre y se muestra junto con las cadenas "Hola" y "bienvenido".

Es importante tener en cuenta que la funci√≥n input devuelve siempre un valor de tipo string. Funciones definidas por el usuario¬∂

Adem√°s de las funciones nativas de Python, es posible definir nuestras propias funciones. En Python, de manera general, una funci√≥n se define siguiendo la estructura mostrada a continuaci√≥n:

def nombre_fun(arg1, arg2, ..., argN):
    # Cuerpo de la funci√≥n
    # .
    # .
    # .
    return val1, val2, ..., valN

Donde def es una palabra que debe anteceder siempre a la definici√≥n de una funci√≥n, nombre_fun es el nombre que se asignar√° a la funci√≥n, entre par√©ntesis y separados por comas se colocan los nombres de los argumentos de entrada, los dos puntos se colocan despu√©s de cerrar el par√©ntesis e indican que ah√≠ termina el encabezado de la funci√≥n y comenzar√° el cuerpo de la misma, aqu√≠ se colocar√°n todas las instrucciones que deber√°n realizarse; la palabra reservada return sirve para indicar los valores a devolver, mismos que se colocar√°n separados por comas.

Vamos a definir una funci√≥n llamada saluda, la cual recibe un nombre (string) y devuelve un saludo (string) formado mediante concatenaci√≥n:

def saluda(nombre):
    s = "Hola " + nombre + ", bienvenido."
    return s

print(saluda("Jorge"))

Hola Jorge, bienvenido.

Lo √∫nico que hace la funci√≥n anterior es tomar un string como argumento y unirlo a algunas cadenas ya establecidas dentro de la funci√≥n.

Veamos ahora c√≥mo definir una funci√≥n que recibe como argumento un entero y devuelve un valor l√≥gico que indica si este es par.

def espar(n):
    if n%2 == 0:
        s = True
    else:
        s = False
    return s

print(espar(2))
print(espar(5))
print(espar(10))

False

La funci√≥n mayor recibe dos valores num√©ricos y determina cu√°l es el mayor de ambos mediante una comparaci√≥n con la sentencia if.

Funciones con una cantidad de par√°metros indeterminada¬∂

En ocasiones el n√∫mero de par√°metros que deber√° recibir una funci√≥n no puede ser algo fijo. Las definiciones de funci√≥n en Python tienen la flexibilidad de poder recibir una cantidad variable de argumentos de entrada.

Para ejemplificar esto, vamos a crear una funci√≥n llamada promedio que calcule el promedio de una cierta cantidad de n√∫meros pasados como argumentos:

def promedio(*numeros):
    suma = 0
    k = 0
    for n in numeros:
        suma += n
        k += 1
    return suma/k

print(promedio(10,5))
print(promedio(10,50,40,80,20,100))
print(promedio(5,15,10,5))

7.5
50.0
8.75

Observe que lo √∫nico que hacemos es que al nombre del par√°metro le anteponemos un asterisco, esto le indica a Python que la cantidad de argumentos de entrada es indeterminada, en principio. Claro est√°, que el manejo posterior de la informaci√≥n es algo que el programador debe tener en cuenta. Dentro del cuerpo de la funci√≥n se debe considerar que el par√°metro numeros ser√° una tupla cuya cantidad de elementos depender√° de la cantidad de argumentos ingresados.

Funciones y los argumentos con nombre¬∂

Una funci√≥n en Python se puede mandar a llamar pas√°ndo los argumentos de manera posicional, es decir, en el orden que fueron definidos en la funci√≥n, o bien, haciendo uso del nombre del par√°metro correspondiente al argumento que se introduce, por ejemplo:

def cuenta_cuantas(frase, letra):
    k = 0
    for car in frase:
        if car is letra:
            k += 1
    return k

print( cuenta_cuantas("hola mundo", "o") )
print( cuenta_cuantas(frase="hola mundo", letra="o") )
print( cuenta_cuantas(letra="o", frase="hola mundo") )

2
2
2

La funci√≥n cuenta_cuantas devuelve el n√∫mero de presencias de una determinada letra en una frase. Observe las tres formas en que la ejecutamos, todas son equivalentes. En la primera se pasan los argumentos de forma posicional, en la segunda y tercera se utilizan los argumentos con nombres, note que en este caso el orden en que los argumentos son pasados, es indistinto.

En la definici√≥n de funciones es posible tambi√©n especificar que se pasar√°n ciertos argumentos con nombre sin necesidad de escribirlos de manera expl√≠cita. Observe la siguiente funci√≥n:

def muestra_puntos(**personas):
    for persona in personas.items():
        print(persona[0] + " tiene " + str(persona[1]) + " puntos")

muestra_puntos(Jorge=8, Paty=10)
print(30*"=")
muestra_puntos(Ana=6, Carlos=9, Victor=4, Daniela=8)

Jorge tiene 8 puntos
Paty tiene 10 puntos
==============================
Ana tiene 6 puntos
Carlos tiene 9 puntos
Victor tiene 4 puntos
Daniela tiene 8 puntos

Vea que la definici√≥n de la funci√≥n muestra_puntos incluye un par√°metro llamado **personas, esos dos asteriscos antes del nombre del par√°metro, indican que no se tiene predeterminado el n√∫mero de argumentos que se pasar√°n, pero adem√°s, indica que cada argumento a introducir deber√° ser un argumento con nombre. Dentro del cuerpo de la funci√≥n el par√°metro **personas es un diccionario cuyas claves son los nombres de los argumentos y los valores corresponden a cada valor asignado al argumento.
